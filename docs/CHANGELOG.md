# LEOLRS0-3 更新与修复日志

本文档是系统版本、功能更新、Bug 修复和回测口径变更的本地永久记录。后续所有版本的更新内容都应追加在本文件中，不能覆盖或删除旧版本记录。

## v0.2.3 - 2026-05-11

### 1. 功能优化：合并市场与交易模式时间轴

今日信号页的交易窗口显示改为 24 小时合并时间轴。NZX、ASX、US 三个市场不再分别显示独立进度条，而是在同一条市场时间轴中展示：

- NZ 使用黑色。
- 澳大利亚 / ASX 使用红色。
- 美国 / US 使用蓝色。
- 多个市场重叠的时段使用交替粗斜线填充，方便一眼识别重合交易窗口。

交易模式时间轴与市场时间轴上下并行，共用同一个 24 小时时间尺度。

### 2. 功能优化：今日信号页交易模式切换

今日信号页新增交易时间轴模式选择。默认模式改为 `NZ 盘末 / 美股开盘`，并且交易模式时间轴只显示当前选中的模式，不再同时展示多种操作口径。

NZ 盘末 / 美股开盘模式下，时间轴上的操作点改为短标签，完整操作说明移动到时间轴下方的自适应清单，避免多个操作点文字互相重叠。

### 3. 功能新增：市场和操作倒计时

合并时间轴下方新增倒计时区域：

- 如果有当前正在交易的市场，显示当前市场收盘倒计时。
- 如果当前模式有即将到来的操作点，显示当前模式操作时间倒计时。
- 进入 3 小时窗口后，倒计时卡片改为红底白字。
- 如果没有当前市场收盘时间和当前模式操作时间，则显示下一个市场开盘倒计时。

### 4. UI 优化：手机布局和交易模式标记

合并时间轴进一步优化了手机屏幕显示：

- 市场时间轴标题放在市场轴上方。
- 交易模式时间轴的时间标记和标题放在交易轴下方，避免与轴上内容重叠。
- `Now` 标记只保留一个，并放在交易模式时间轴下方的时间标记区域。
- NZ 盘末 / 美股开盘模式的三个操作点使用不同颜色：NZX 收盘为橙色，美股开盘为蓝色，美股收盘为紫色。
- 下一交易日模式同步采用同一套设计语言，使用绿色操作标记。
- 操作点前的 3 小时提示色块改为全高色块，透明度 50%，并与对应操作点颜色一致。
- 操作说明改为短句，降低手机屏幕上的文字拥挤。

## v0.2.2 - 2026-05-10

### 1. 功能新增：调试模式参数扫描升级为 50% 完整建议报告

根据本轮对话需求，“调试模式：参数扫描”从原来的 20% 敏感性扫描升级为 50% 扫描。默认扫描倍率调整为 50%、75%、100%、125%、150%。系统仍保留当前全区间扫描，并继续以当前配置为主基准线进行对比。

扫描结果新增默认配置基准线对比。每个扫描结果现在同时展示相对当前配置的收益差，以及相对默认配置的收益差，方便判断当前配置和默认配置之间的实际改进。

### 2. 功能新增：目标日期、时间窗口和排序目标

参数扫描区新增以下输入和结果：

- 目标日期：用户可以选择一个关键市场日期作为优化锚点。
- 时间窗口：用户可以设置目标日前和目标日后的月份范围，系统会围绕目标日期执行局部窗口回测。
- 排序目标：支持按策略总收益、CAGR、Sharpe、最大回撤、年化波动、调仓次数排序。
- 目标日期参数建议表：在全区间建议之外，额外输出目标窗口内的参数建议。

该设计用于支持针对不同市场目标设计不同交易模式，例如防守模式、反弹模式、趋势延续模式和震荡过滤模式。

### 3. 功能新增：扫描结果曲线和 PDF 报告

参数扫描完成后，界面会输出：

- 全区间对比净值曲线：当前配置、默认配置、最佳单参数、最佳统一参数。
- 目标窗口对比净值曲线：当前配置、默认配置、最佳单参数、最佳统一参数。
- 全区间单参数扫描折线。
- 目标窗口单参数扫描折线。

历史回测 PDF 现在会包含净值曲线、仓位曲线、参数扫描建议、目标日期参数建议，以及上述扫描曲线和折线。

### 4. 功能新增：参数结果显示 UI 命名

参数扫描输出不再只显示内部参数路径，也会显示该参数在 UI 中对应的名称。UI 命名会根据当前语言切换：

- 中文模式显示中文 UI 名称，例如“短期均线”“最大等效仓位”“low 系数”。
- 英文模式显示英文 UI 名称，例如 “Short moving average”“Maximum equivalent exposure”“low multiplier”。

建议表、参数范围表、逐个测试表、统一测试表、PDF 参数建议，以及扫描折线图都已接入 UI 命名。

### 5. 功能新增：英文模式下显示英文更新历史

“设置总览”页面在中文模式下继续读取 `docs/CHANGELOG.md`。当界面语言为英文时，更新历史会读取 `docs/CHANGELOG.en.md`。中文更新日志仍作为主要记录，英文文件作为翻译展示。

### 6. UI 修复：顶部操作按钮对齐

以下按钮过去在同一行中比日期输入或其他控件略高，现在通过统一的按钮对齐辅助函数增加顶部留白，使它们与相邻控件在视觉上对齐：

- 更新市场健康度
- 运行回测
- 今日信号中的更新今日信号按钮
- 设置总览中的保存当前设置
- 设置总览中的另存为配置文件包

代码中没有单独名为“获取今日数据”的按钮；当前今日信号页的数据获取和信号更新由“更新今日信号”按钮完成。本次对齐已覆盖该位置。

### 7. 修复：参数扫描遵循策略设置上限

调试模式中的参数扫描现在会遵循 UI 和策略设置中的上限，避免 50% 扫描把参数推到非法范围。例如：

- 最大等效仓位不会超过 300%。
- 趋势仓位类参数不会超过当前策略最大等效仓位，且最高不超过 300%。
- 最小调仓阈值不会超过 UI 上限 30%。
- VIX 系数不会超过 UI 上限 5.0。
- 均线窗口和确认天数不会超过各自 UI 上限。

统一扫描结果的备注也会使用裁剪后的真实测试值，避免备注显示未裁剪值而实际回测使用裁剪值。

### 8. 验证记录

验证命令：

```bash

uv run pytest -q
```

当前验证结果：58 个测试全部通过。

## v0.2.1 - 2026-05-10

### 1. 功能新增：历史回测支持每周追加资金

历史回测设置区新增“每周追加资金”输入框。该参数与初始资金并列，用于模拟定投或持续入金场景。

具体计算口径如下：

- 可见回测区间第一条记录只使用初始资金，不在同一条记录额外追加资金。
- 从第二个可见交易周开始，每个新交易周的第一条交易日记录开始前追加指定金额。
- 追加资金同时进入策略净值曲线、S&P 500 买入持有曲线、3 倍 S&P 500 买入持有曲线、S&P 500 120 日择时曲线，以及三倍持有跌破 120 日均线转现金曲线。
- 追加资金影响组合净值金额和后续调仓时用于仓位上限检查的组合规模。
- 收益率、CAGR、Sharpe 等指标仍按每日收益序列计算，避免把现金流本身误计为投资收益。

本功能新增 `backtest.weekly_contribution` 配置项，默认值为 `0.0`。默认配置和所有现有配置包均已补齐该字段，保证保存后可以被 TOML 重新读取。

### 2. 检查：设置侧边栏模块保存与稳定性

本版本对“设置配置”侧边栏做了结构和稳定性检查，重点确认以下方面：

- 各模块的配置项会写入当前内存中的配置对象，并可通过“设置总览”页面保存到当前配置文件或另存为新的配置文件包。
- VIX 风险模块、回撤风险模块、区段无新高锁仓模块、趋势质量模块在关闭状态下默认收起，启用后可展开调整参数。
- 区段无新高锁仓模块的三个参数均有明确边界，避免出现非法负数或超出最大仓位的锁定比例。
- 风险曲线分段边界使用递增下限，避免用户将后一个边界设置得低于前一个边界导致规则无效。
- 仓位类输入会根据基础最小/最大仓位做边界限制，避免保存出明显无效的仓位值。
- 所有配置文件均通过读取、字段检查和 TOML 序列化检查。
- Streamlit 应用已在本地启动并返回 HTTP 200，启动日志未出现运行时报错。

后续如果继续增加侧边栏模块，应同步检查：默认值是否完整、保存后 TOML 是否可重新读取、数值输入是否有稳定 key 或稳定标签、上下界变化是否会造成 Streamlit 控件值被意外夹紧。

## v0.2.0 - 2026-05-10

### 1. 修复：回测区间起点导致信号状态失真

在历史回测中发现，同一套 LEO 配置下，较短回测区间的调仓次数可能反而高于较长回测区间。例如从 2000-01-01 回测到当前日期时调仓次数低于从 2010-01-01 回测到当前日期的结果。这不是策略逻辑本身的真实差异，而是数据窗口截断导致的状态失真。

根因是系统过去直接使用用户选择的回测起点作为价格下载和信号计算起点。这样会使长期均线、无新高计数、回撤窗口、趋势质量窗口、120/200 日均线健康度等依赖历史上下文的模块在回测起点附近失去必要的预热数据。2010 起点的回测不会继承 2009 年以前的状态，因此 2010 年以后生成的信号不再是 2000 起点回测结果的子集，最终造成调仓次数、仓位路径和收益曲线不一致。

修复后，系统会根据当前策略配置自动计算所需的历史预热天数，并在内部提前下载足够的数据。回测输出仍只展示和统计用户选择的可见区间，预热区间只用于计算信号状态，不计入净值、收益、调仓次数或指标统计。

### 2. 修复：回测默认参考线的第一天收益泄漏

检查历史回测中的默认参考线后，确认 S&P 500 买入持有、3 倍 S&P 500 买入持有、S&P 500 120 日择时、三倍持有跌破 120 日均线转现金等曲线也受到时间窗口问题影响。

动态预热修复后，这些参考线能够正确使用预热历史计算 120 日均线和择时状态，但它们在可见回测区间的第一天仍可能把“预热区间最后一个交易日到回测起点当天”的涨跌幅计入收益。这会让参考线在回测起点第一天不是从初始资金开始，而是提前吃到或承担了一天不属于用户选择区间的收益。

修复后，所有回测曲线在可见区间第一条记录统一以初始资金开始，当日收益记为 0。预热区间只决定进入可见区间时的信号状态和持仓状态，不贡献任何可见区间收益。受影响并已统一修复的曲线包括：

- 策略净值曲线
- S&P 500 买入持有基准
- 3 倍 S&P 500 买入持有基准
- S&P 500 120 日择时曲线
- 三倍持有：跌破 120 日均线转现金曲线

### 3. 修复：今日信号和市场健康度同样存在起点截断问题

同类问题也存在于“今日信号”和“市场健康度”两个栏目。过去这两个栏目直接使用界面中选择的数据起始日期下载数据并计算指标。如果用户选择的起始日期过晚，最新信号会缺少长期均线、趋势质量、无新高、回撤等模块所需的历史上下文，市场健康度也可能因为 MA120/MA200 预热不足而显示错误状态。

修复后，“今日信号”和“市场健康度”会使用与回测相同的动态预热逻辑。界面仍以用户选择的日期作为展示语义，但内部会提前下载足够历史数据来保证最新信号和健康状态计算完整。

### 4. 功能调整：无新高锁仓模块升级为区段无新高锁仓模块

原来的无新高锁仓模块只判断连续多少天没有创出历史新高或窗口新高，并且锁定仓位固定为 100%。这会导致两个问题：第一，模块语义不够明确，无法区分“观察多久没有新高”和“创多少日新高”；第二，锁定仓位比例不可调整，无法适配不同配置包的风险偏好。

本版本将其升级为“区段无新高锁仓模块”。新逻辑为：如果指定观察期内没有出现过“指定日期区段的新高”，则触发锁仓，并将目标等效仓位限制到配置的锁定仓位比例。

新增和保留的可调参数如下：

- 锁定仓位比例：触发锁仓后允许的最高目标等效仓位，默认 100%，允许用户调整。
- 无新高观察天数：判断最近多少个交易日内是否出现过区段新高。
- 日期区段新高：定义“创多少日新高”，例如 200 表示创 200 日收盘新高。

LEO 配置包当前设置为 200 日观察期、200 日新高区段、100% 锁定仓位。

### 5. UI 改进：风控模块支持折叠

策略设置界面中的多个可关闭模块现在使用可展开和可收起的信息区块承载，减少设置页纵向长度，便于只关注当前启用或正在调整的模块。

支持折叠的模块包括：

- VIX 风险模块
- 回撤风险模块
- 区段无新高锁仓模块
- 趋势质量模块

模块默认展开状态会参考当前配置。如果模块已启用，进入设置页时会优先展开；如果未启用，则默认收起。

### 6. 设置总览：版本号和日志展示

“设置总览”页面现在在当前设置内容的末端显示系统版本号和更新修复日志。日志不再写死在界面代码中，而是读取本地文件 `docs/CHANGELOG.md`。页面中的日志区域使用固定高度和滚动条显示，避免版本记录随着时间增长而无限拉长设置页。
